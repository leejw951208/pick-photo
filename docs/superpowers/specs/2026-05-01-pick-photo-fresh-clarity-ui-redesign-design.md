# Pick Photo Fresh Clarity UI 리디자인 설계

## 목표

Pick Photo 모바일 앱의 첫 화면, 얼굴 검토 화면, 결과/실패/진행 상태를 `Fresh Clarity` 방향으로 리디자인한다. 현재 기능 중심의 카드형 화면을 밝은 민트/블루 기반의 신뢰감 있는 사진 앱 톤으로 바꾸되, 기존 제품 요구사항인 원본 사진 직접 얼굴 선택, 확대/이동 보조, 하단 선택 요약, 선택된 얼굴만 생성 규칙은 유지한다.

## 선택된 디자인 방향

사용자는 2026년 5월 1일 리디자인 옵션 목업 중 `C. Fresh Clarity`를 선택했다.

이 방향은 다음 특징을 가진다.

- 밝은 민트와 블루를 중심으로 한 깨끗한 모바일 앱 톤.
- 큰 상태 헤더와 단계 칩으로 업로드, 얼굴 선택, 결과 확인 흐름을 명확히 표현.
- 결과 검토와 실패 상태에서 사용자가 다음 행동을 빠르게 이해할 수 있는 대비 높은 UI.
- 사진과 얼굴이라는 민감한 정보를 다루므로 장식보다 선택 확신, 상태 명확성, 접근성을 우선.

## 제품 기준

이 리디자인은 `PRD.md`의 다음 기준을 보존한다.

- 사용자는 사진 한 장을 선택해 업로드 흐름을 시작할 수 있어야 한다.
- 사용자는 인식된 얼굴을 명확히 확인하고 선택할 수 있어야 한다.
- 선택되지 않은 얼굴은 결과 생성 대상이 되면 안 된다.
- 생성 전에 어떤 얼굴이 선택되었는지 이해할 수 있어야 한다.
- 실패 상태에서는 다시 시도할 수 있는 다음 행동이 보여야 한다.
- 사진, 얼굴, 생성 결과는 민감한 정보로 취급되어야 한다.

## 비목표

- API, 백엔드, AI 서버, 데이터베이스 계약 변경은 포함하지 않는다.
- 국가별 증명사진 규격 선택, 보정 도구, 배경색 편집, 다운로드 권한 정책은 포함하지 않는다.
- 새 이미지, 외부 폰트, 새 Flutter 패키지 추가는 포함하지 않는다.
- 보관 기간, 삭제 방법, 공식 규격 보장처럼 아직 결정되지 않은 정책을 UI 문구로 약속하지 않는다.

## 화면 설계

### 앱 테마

- 배경은 차가운 화이트와 옅은 블루 계열을 사용한다.
- Primary는 딥 네이비, Accent는 민트, Secondary는 선명한 블루로 구성한다.
- 카드 반경은 기존 8px보다 넓히되, 앱 전체가 과도하게 둥근 장식처럼 보이지 않도록 주요 패널은 18-24px 범위로 제한한다.
- 그림자는 약하게 사용해 밝은 화면에서도 계층이 보이게 한다.

### 시작 상태

- 상단 헤더는 앱의 목적을 더 감각적으로 보여준다.
- “선택한 얼굴만 생성”과 “저장 전 확인” 메시지는 유지하되 더 가벼운 신뢰 배지로 표현한다.
- 안내 목록은 반복 카드 느낌을 줄이고, 흐름을 한눈에 스캔할 수 있는 타일 형태로 정리한다.

### 얼굴 검토 상태

- 기존 원본 사진 직접 선택 방식을 유지한다.
- `FaceSelectionCanvas`는 밝은 화면 안에서도 사진 영역이 명확히 보이도록 푸른 톤의 컨테이너와 더 선명한 선택/제외/확인 필요 라벨을 사용한다.
- 확대, 축소, 맞춤, 원본 버튼은 텍스트가 긴 버튼보다 작은 보조 컨트롤처럼 보이게 정리한다.
- 하단 선택 요약은 선택 수, 전체 수, 전체 선택, 선택 초기화, 생성 액션을 유지하되 CTA 우선순위를 더 명확히 한다.

### 결과 상태

- 결과 카드는 작은 리스트 느낌보다 검토 가능한 미리보기 카드 느낌으로 개선한다.
- 결과 URL은 디버그 정보처럼 보이지 않도록 보조 정보로 낮추고, 핵심은 “선택 얼굴 결과 확인”에 둔다.
- 결과가 네트워크 이미지로 표시되지 않을 때도 대체 UI가 깨끗하게 보이도록 한다.

### 진행/실패 상태

- 진행 상태는 빈 화면 중앙 스피너 대신 현재 작업 메시지를 담은 패널로 보여준다.
- 실패 상태는 사용자에게 다른 사진으로 재시도할 수 있음을 명확히 보여주며, 불필요한 정책 약속은 하지 않는다.

## 접근성

- 기존 `Semantics` 기반 얼굴 선택 접근성을 유지한다.
- 색상만으로 선택 상태를 전달하지 않고 `선택됨`, `제외됨`, `확인 필요` 텍스트를 유지한다.
- 버튼과 얼굴 선택 터치 영역은 최소 44px 조작 영역을 유지한다.
- 텍스트는 Flutter `Theme.of(context).textTheme` 기반을 유지하고 고정 크기 사용을 최소화한다.

## 구현 영향

변경 대상은 Flutter 앱 UI에 한정한다.

- `apps/mobile/lib/main.dart`: ColorScheme, ThemeData, 버튼/카드/앱바 톤 조정.
- `apps/mobile/lib/features/photo_flow/photo_flow_screen.dart`: 헤더, 안내, 검토 패널, 결과 카드, 진행/실패, 선택 요약 스타일 조정.
- `apps/mobile/lib/features/photo_flow/face_selection_canvas.dart`: 사진 캔버스, 얼굴 라벨, 확대 컨트롤 스타일 조정.
- `apps/mobile/test/widget_test.dart`: 시작 화면 문구나 주요 UI 기준 변경 시 갱신.
- `apps/mobile/test/photo_flow_screen_test.dart`: 기존 기능 테스트가 새 문구에도 통과하도록 필요한 기대값 갱신.

## 보안과 개인정보

이 작업은 새 데이터 수집, 전송, 저장, 다운로드를 추가하지 않는다. 기존 업로드/생성 흐름과 API 호출 방식은 바꾸지 않는다. UI 문구는 “선택한 얼굴만 생성”과 “저장 전 확인” 수준의 현재 제품 기준만 표현하고, 보관 기간 또는 삭제 보장처럼 미정인 정책을 약속하지 않는다.

## 검증 기준

- Flutter 위젯 테스트가 통과한다.
- Dart 포맷이 통과한다.
- 업로드 전, 얼굴 검토, 선택 없음, 선택 후 생성, 실패, 결과 상태 테스트가 기존 기능 의미를 유지한다.
- 텍스트가 모바일 폭에서 주요 버튼과 패널을 밀어내거나 겹치지 않는다.

## 스펙 자체 점검

- 미완성 표기 점검: 미완성 표기나 빈 섹션은 없다.
- 내부 일관성 점검: 선택된 방향은 `Fresh Clarity` 하나로 고정되어 있고, 기존 직접 얼굴 선택 UX와 충돌하지 않는다.
- 범위 점검: Flutter UI 표현 변경에 집중하며 API, AI, DB, 개인정보 정책 확정은 범위 밖이다.
- 모호성 점검: 구현 대상 파일과 비대상 범위를 명시했다.
